> 面向：**微信小程序工程落地**（代码、域名、云托管、提审）。  
> 配套后端：仓库根目录 **`backend/`**（Express，`Dockerfile` 已就绪）。  
> 云托管服务名示例：**`express-17qb`**（以你控制台实际名称为准）。

**本项目当前云托管默认域名（已与 `utils/apiConfig.js` 对齐）**：

- 根地址：`https://express-17qb-243105-7-1419714752.sh.run.tcloudbase.com`（无末尾 `/`）
- 健康检查：`curl https://express-17qb-243105-7-1419714752.sh.run.tcloudbase.com/health`
- 公众平台 **request 合法域名** 主机名：`express-17qb-243105-7-1419714752.sh.run.tcloudbase.com`

## 1. 上线前架构（你需要清楚的三件事）

| 模块 | 说明 |
|------|------|
| **小程序前端** | 本目录 `miniprogram/`，在微信开发者工具 **上传** → 公众平台 **提交审核** → **发布**。 |

| **数据与密钥** | `WECHAT_APP_SECRET`、数据库密码等只放在 **云托管环境变量** 或 **密钥管理**，**不要**写进 Git。 |

## 2. 域名要不要买？

| 场景 | 是否需要自购域名 |
|------|------------------|
| **使用云托管默认域名** | **不必购买**。形如 `https://xxxx.sh.run.tcloudbase.com`，HTTPS 由平台提供，可直接用于 **request 合法域名**（按公众平台规则完成校验即可）。 |
| **希望品牌统一、长期稳定、自有备案** | **建议购买**并完成 **ICP 备案**（若面向大陆用户且政策要求），在 **云托管 / 云开发** 中绑定 **自定义域名**，再在小程序后台配置该域名。 |
| **仅开发联调** | 可用本机 `http://127.0.0.1:3000`，开发者工具勾选 **不校验合法域名**；**真机体验/正式版必须用 HTTPS 合法域名**。 |

**结论**：用云托管默认域名即可先上线；有品牌与合规需求时再上自有域名。

## 4. 后端部署：微信云托管（服务示例：`express-17qb`）

以下步骤在 **微信云开发控制台 → 云托管** 中操作；服务名称以你创建的为准（文中以 **`express-17qb`** 为例）。

### 4.1 创建 / 进入服务

1. 打开 [微信云开发 / 云托管控制台](https://console.cloud.tencent.com/tcb)（从开发者工具 **云开发** 入口也可进）。
2. 选择与环境对应的小程序 / 云开发环境。
3. 进入 **云托管 → 服务列表**，找到服务 **`express-17qb`**（或你新建的服务）。

### 4.2 流水线（从 Git 构建镜像）

1. 进入该服务 → **服务设置 → 流水线**。
2. **代码源**：如 **GitHub**，先完成 **授权**（解决「代码权限未授权或授权过期」）。
3. **代码仓库**：选择存放本仓库的仓库（需包含 `backend/` 目录）。
4. **分支**：如 `main`（与你日常推送分支一致）。
5. **目标目录**：填 **`backend`**（与仓库内 `backend/Dockerfile` 所在目录一致）。
6. **端口**：与云托管「容器端口」一致即可（常见 **80** 或 **3000**）。本仓库后端读取 **`process.env.PORT`**，平台注入的端口与这里一致即可。
7. **Dockerfile**：选择 **有**；**Dockerfile 名称** 填 **`Dockerfile`**。
8. 按需开启 **推送触发**（推分支自动构建）。
9. **保存**。

### 4.3 部署发布

1. 进入 **部署发布**（或等价入口），选择最新构建成功的镜像 **发布**。
2. 等待实例 **运行中**，在 **服务设置 → 基础信息** 中确认 **公网访问** 已开启，并复制 **默认公网域名**（`https://...sh.run.tcloudbase.com`）。

### 4.4 健康检查（建议）

在 **服务设置** 中将健康检查路径设为 **`/health`**，方法 **GET**。部署后本机执行：

```bash
curl https://你的云托管默认域名/health
```

应返回类似：`{"ok":true,"service":"adslot-backend",...}`。

### 4.5 环境变量（服务设置 → 基础信息）

至少配置（名称与 `backend/.env.example`、后端代码一致）：

| 变量 | 说明 |
|------|------|
| `WECHAT_APP_ID` | 小程序 AppID |
| `WECHAT_APP_SECRET` | 小程序 AppSecret |
| `WECHAT_ADMIN_OPENIDS` | 可选；生产环境 **中台管理员** 的 openid，英文逗号分隔 |

说明：

- 当前仓库后端默认仍使用容器内 **`data/db.json`** 持久化；若你在控制台配置了 `MYSQL_*`、`COS_*`，需 **自行改后端代码** 才会生效，否则可暂不配。
- 修改环境变量后一般需 **重新部署/重启实例** 生效（以控制台提示为准）。

### 4.6 资源与扩缩容（你截图中的配置）

- **最小副本为 0**：冷启动首次请求可能略慢，属正常现象；若要求始终快速响应，可将最小副本调为 **1**（成本略增）。
- **CPU / 内存**：按访问量再调；初期 1C2G 通常可支撑演示与小流量。

## 5. 小程序侧：配置合法域名

1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → **开发 → 开发管理 → 开发设置**。
2. **服务器域名 → request 合法域名**：
   - 添加云托管 **HTTPS 默认域名**（**无路径、无末尾 /**）。
3. 按页面提示完成 **域名校验**（下载校验文件或按当前平台流程；若使用自定义域名，按 DNS 与证书要求配置）。

**注意**：真机与体验版 **默认校验域名**；未配置的域名会请求失败。

---

## 6. 小程序代码：上线前必改（`miniprogram/utils/apiConfig.js`）

1. **`useServer`**：保持 **`true`**（走云托管 API）。
2. **`baseUrl`**：改为云托管 **HTTPS 根地址**，例如：  
   `'https://xxxx.sh.run.tcloudbase.com'`  
   **不要** 末尾斜杠 **`/`**。
3. **去掉或注释** 仅用于本机调试的 **`secondaryBaseUrl`**（避免误连局域网 IP）；若团队仍需本地调试，可用 Git 分支或本地改配置，**勿把内网地址提交为默认上线配置**。
4. **`debugApiLog`**：正式上架建议设为 **`false`** 或保持 `undefined` 由环境判断，避免正式包刷屏日志。

修改后重新 **编译**，在开发者工具用 **真机预览** 测一遍登录与主要流程。

## 附录：复制即用模板（填写 `YOUR_*`）

把下面 **占位符** 换成你在云托管控制台里看到的真实值即可；**不要**把含密钥的截图或 `.env` 提交到公开仓库。

### A. 从控制台抄三项

| 占位符 | 含义 | 你从哪里抄 |
|--------|------|------------|
| `YOUR_CLOUD_HOST` | **仅主机名**，不含 `https://`、不含路径、不含末尾 `/` | **云托管 → 服务 `express-17qb` → 服务设置 → 基础信息 → 公网访问** 里的域名，例如 `https://abc-xxx.sh.run.tcloudbase.com` → 只取 `abc-xxx.sh.run.tcloudbase.com` |
| `YOUR_PIPELINE_PORT` | 流水线里填的 **端口** | **服务设置 → 流水线 → 端口**（常见：`80` 或 `3000`） |

说明：

- 后端代码使用 **`process.env.PORT`**，云托管会注入与容器端口一致的 `PORT`，**你无需在小程序里写端口号**；`baseUrl` 只用 **HTTPS 默认域名**（**不带** `:80` / `:443`，浏览器与小程序默认端口即可）。
- 若流水线端口为 **80**，`curl` 测健康检查仍写：`curl https://YOUR_CLOUD_HOST/health`（无需写 `:80`）。

### B. 自检命令（替换主机名后终端执行）

```bash
curl https://YOUR_CLOUD_HOST/health
```

本项目可直接执行：

```bash
curl https://express-17qb-243105-7-1419714752.sh.run.tcloudbase.com/health
```

期望返回含 `"ok":true` 与 `"service":"adslot-backend"` 的 JSON。

### C. 公众平台「request 合法域名」

在 **开发设置 → 服务器域名** 中，**request 合法域名** 一般填写为：

```text
YOUR_CLOUD_HOST
```

（与微信后台表单项一致即可；若后台要求带 `https://` 前缀，按界面提示为准。）

### D. `miniprogram/utils/apiConfig.js` 上线用片段（复制后替换）

仓库内 **已默认填写** 当前云托管地址；其它环境可改为占位符：

```javascript
/**
 * 上线：替换 YOUR_CLOUD_HOST 后保存；勿提交本仓库到公开库若含真实域名争议。
 */
module.exports = {
  useServer: true,
  /** 正式版建议 false，避免控制台刷屏 */
  debugApiLog: false,
  /** 云托管 HTTPS 根地址，无末尾 / */
  baseUrl: 'https://YOUR_CLOUD_HOST',
  /** 上线可留空字符串，不再用局域网备用 */
  secondaryBaseUrl: '',
  preferLoopbackFirst: true,
};
```

本地调试时仍可临时改回 `http://127.0.0.1:3000` 并填 `secondaryBaseUrl`，**合并到主分支前**请恢复为云托管 HTTPS 配置。

## 7. 上传代码与提审、发布

1. **微信开发者工具** → 右上角 **上传**，填写版本号与项目备注。
2. 登录 **微信公众平台 → 管理 → 版本管理**：

   - 确认无误后 **提交审核**，填写 **功能页面与测试账号**（若审核需要）。
3. 审核通过后 **发布**；用户使用的即为 **正式版**。

## 9. 回滚与问题排查

| 现象 | 可排查方向 |
|------|------------|
| 真机请求失败 | 合法域名、是否 HTTPS、`baseUrl` 拼写、云托管实例是否运行 |
| 登录失败 | `WECHAT_APP_SECRET`、AppID 是否同一小程序、后端日志 |
| 审核被拒 | 类目、隐私协议、测试路径说明是否完整 |

版本回滚：公众平台 **版本管理** 可选上一线上版本回退；云托管可在 **部署发布** 选用历史镜像。
